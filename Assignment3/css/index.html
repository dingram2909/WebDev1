Cascading Style Sheets

A long time ago in a galaxy far, far away there existed a standard of HTML which used inline font tags to format text. And inline style tags to format elements on a page. These were dissident, unruly times, where the elders of the internet bickered and squabbled on a daily basis. That was until, one day, a valiant knight called W3C came along and, with his trusty blade named CSS and his sturdy shield called the W3 Validator, vanquished these archaic HTML elements to the realms of darkness and obscurity, never to be heard from again. The elders rejoiced and adapted their kingdoms (read: web browsers) to acknowledge, and accept, the existence of the new standards the W3C set out upon his victory. The lands of the World Wide Web are now calmer than they once were, though there are some who seek to recall the old ways, and use them to destroy all that is held dear. But we will not tolerate it. We will create our style with CSS, and we will identify and protect ourselves against deprecated tags using the W3 Validator.

Long story short: CSS good, inline formatting bad. And it's not likely to change in the future. Unless Napoleon takes control. Sorry, once again I've gone off on a bit of a tangent. I promise there will be no more sci-fi or literature references on this web site. Well, I would, but I can't really do that as I just type what comes to mind. Ho-hum. CSS!

CSS is the main component of web development used to style websites. CSS is powerful, yet can be incredibly subtle. CSS can change your boring, plain web page (sorry, but at the moment it is) into something spectacular. People with a lot more creativity than myself can make things look fantastic. I can make web sites which look good, and are functional, but not especially artistic or pretty (for examples of awesome websites developed by others, visit the CSS Zen Garden).
