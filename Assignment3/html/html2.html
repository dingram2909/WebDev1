Essential Tags

Throughout the course of your web development you will notice that there are certain tags which are used a lot more than others. These are things like your heading tags (h1 to h6, used, remarkably, for headings), your paragraph tags (p, used for blocks of text), and your anchor tags (a, used for hyperlinks, or occasionally crude styling).

Generally most tags share at least four main attributes, some can have more than the generic four, but some are restricted to just four. These attributes, which don't have to be used, are class; id; style; and title. Your class attribute is used for assigning a CSS defined class to your content. This class would contain certain styling information such as the font-face to be used or how bright to make the pink you want to use. The id attribute works in a similar manner, although you can only assign an ID to one tag. So these are generally used for your page division (div) tags.

The style attribute is used for applying style to tags. You don't generally see these anymore as they have been largely made redundant by CSS; we won't be using them in this website. Neither will we be using the title attribute, which gives a title to the tag it is included in. This is used to add more accessibility to the website, it is very useful to screen readers.

Page Division Tags

One of the core tags in web development is the page division tag, div, this is used to allocate page divisions. Essentially how it works is that it separates the page up into easily manipulated blocks. At the moment we haven't learnt how to manipulate these blocks, but later we'll be using them quite extensively, so put some into your document.

I like to put my entire website in one set of division tags so that I can centre it in the browser, so create a couple of div tags and give them the id of "Website". It is important to note that there are no real accepted naming conventions in terms of class and ID names, as long as you keep the same style across the board you should be ok. I like to create IDs and give them an upper case starting letter. Classes are all lower case names, but this is just personal preference. Feel free to name your IDs and classes how you wish. As long as it's not rude. Well, not too rude. Actually, go ahead and do what you like.

Paragraph Tags

Paragraph tags are probably the most used of all tags within the HTML document. Paragraph tags, written as <p></p> are the containers for any blocks of text within your website. For example, this block of text you're reading now? Contained within p tags. That big introduction at the top there? p tags. That bacon you're eating? p tags. Ok, well, maybe that last one wasn't paragraph tags. But it would be cool, wouldn't it?

If you need a block of text within your web page you sort of have to surround it with paragraph tags. These can be assigned classes, as with all other tags, and can be extensively customised with some simple CSS (yes, we will hopefully get on to that at some point).
