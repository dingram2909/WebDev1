HTML Basics

Every web site uses the same base language, HyperText Markup Language (HTML), however some use an XML (Extensible Markup Language) variant called XHTML. A majority of these also use Cascading Style Sheets (CSS) to further enhance their sites, though that is covered in our CSS section.

HTML can be produced in one of two ways (well, three, but nobody likes JavaScript, so let's avoid that one for now...), these are typing them into a text editor (much like I am doing right now...well, I was when I typed this) or using a bespoke piece of software, such as Adobe Dreamweaver (eew) or Microsoft Expression Web (I actually don't know what this is like, if it's like Front Page then it's icky), to generate the code for you. The former method is my preference, and the preference of most real web developers. This is due to the fact that when you use a WYSIWYG (What You See Is What You Get) development tool, there is an awful lot of unnecessary, and non-standards compliant, code which is injected into your website.

Considering the various standards of HTML, set out by the W3C, is important, but not integral, when developing a website. A shoddily built website will generally still render and be navigatable (my text editor is throwing that up as a spelling error. It isn't), it just might act differently to how you, or your users, expect it to. It also won't validate when you plug it in to the W3 Validator.

So for our tutorial on here I will be talking you through web development using a simple text editor. You can either use your Operating System's default text editor or download something with a little bit more functionality. I would recommend Notepad++ for Windows; TextWrangler for Mac OS; and for Linux I recommend using the basic GEdit with this plugin. However, alternatively you could use Aptana Studio, which I have used in the past. It's quite a nice little tool, but I'm unsure of how it is in its current iteration. I used 1.2, they're now on version 3.4
